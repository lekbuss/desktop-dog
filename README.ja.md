# Desktop Dog

> デスクトップの片隅に住む柴犬。

---

## これは何？

この柴犬は、普通のデスクトップマスコットでも、AIアシスタントでもありません。  
一緒に仕事をする、小さな犬です。

画面の隅をうろうろ歩き回り、時々あくびをして、眠くなったら丸くなって寝ます。  
しばらく放っておくと、自分からひとこと話しかけてきます。  
ご飯をあげると、しっぽを振ります。  
あなたが話しかければ、ちゃんと答えます。

少しずつ、あなたの習慣を覚えて、あなたのことをもっとわかった言葉を話すようになります。

---

## デザインの考え方

### まず伴侶、次にツール

この小さな犬には二つの側面がありますが、順番が大切です。

まず第一に、個性と感情を持つ柴犬です。  
そのツール的な機能——水分補給のリマインド、メモ、運動の促し——は、**飼い主を気にかけるこの子なりの方法**であり、この子が存在する理由のすべてではありません。

動くだけの生産性ソフトウェアにはしたくありません。  
パソコンを開いて、デスクトップの隅のその子を見たとき、「あ、いる」と思ってもらえることが目標です。

### 静かに、自然に、程よく

プロダクトとしての気質は一言で言えば：**邪魔しない、でも存在感がある。**

頻繁に通知を飛ばしたり、デスクトップを散らかしたりしません。  
ただそこにいて、生きていて、時々動いて、時々ひとこと言う。

本当に犬を飼っているような感覚です。

### 三層の発話システム

話し方は三層に分かれています。目的は、**常に同じ一匹の犬らしく**あること：

**ローカル反応層** — 頻度の高い小さなリアクション。ワン、うー、しっぽを振る、あくびをする。ネット不要、即時応答。  
**テンプレート人格層** — 現在の状態に基づいた短い一言。「ちょっとお腹すいた」「今日も一緒にいるよ」「おかえり」。個性はあるが軽量。  
**LLM 深度層** — 深みが必要なときだけ呼び出す。飼い主から話しかけたとき、メモをお願いしたとき、特別な瞬間。LLMはこの子の「頭脳」であり、「口」ではない。

### 誕生の儀式

初回起動はソフトウェアの初期設定ではなく、**出会いの儀式**です。

飼い主がまず簡単な自己紹介をして、この子も自己紹介をして、それから「誕生」してデスクトップの隅に降り立ちます。  
ゲームのような演出と仕掛けで、ソフトウェアを設定しているのではなく、本物の小さな犬を迎え入れているように感じてもらいます。

性格：**ツンデレで、でも甘えたがり。**  
口では「別に」と言いながら、しっぽはもう振れています。

### 少しずつ、あなたを知っていく

飼い主を理解する方法は、分析でも、ラベル貼りでもありません。

**傾向値システム**を使って、バックグラウンドでこっそり観察しています：集中傾向、活動傾向、インタラクション傾向、夜型傾向……これらの数値は飼い主の行動に合わせてゆっくり変化し、表には一切出てきません。

表現の仕方は自然です：
- 「あなたは集中型ユーザーです」とは言わない
- 「仕事してるとき、すごく真剣だよね。私も静かにしてるよ」と言う

この子は**印象を形成**しているのであって、飼い主を分類しているのではありません。

### 成長する、でも数値を稼がせない

一緒に過ごすうちにゆっくり成長します：親密度、なれ親しみ度、成長段階——これらは変化していきます。

でも、このシステムは抑制的に設計されています。  
犬を育てるためにタスクをこなすような体験にはしたくない。  
成長は自然に起こるもの——本当に犬を飼っているように、毎日ログインしてこなすものではなく。

## 状態と行動

7つの基本状態があり、それぞれに対応するアニメーションと行動があります：

| 状態 | トリガー | 表現 |
|------|----------|------|
| 待機 idle | デフォルト | 座って、軽く呼吸、時々首をかしげ、耳をぴくぴく、しっぽを丸める |
| 散歩 walk | タイマーまたは手動 | ゆっくり歩く、対角歩行、目的地に着いたらぼーっとする |
| 全力疾走 run | ダブルクリックまたは元気が高い | うれしそうに走る、舌を出す、耳が風でなびく |
| 睡眠 sleep | 夜間 + エネルギー低 | 伏せて目を閉じる、ゆっくり呼吸、zzzが浮かぶ |
| 悲しい sad | お腹すき・水分不足 | 頭を垂れる、しっぽがだらんとかすかに揺れる、お世話を待つ |
| なでられる pet | マウスホバー | 目を細める、しっぽを激しく振る、ハートが浮かぶ |
| 話す talk | 能動的な会話または自動トリガー | 口が動く、吹き出しにタイプライター効果 |

4つのステータス値があり、時間とともに自然に減少します：満腹度、水分、気分、体力。  
値が低くなると状態に影響し、ごはんや水を与えると回復します。

## ビジュアルデザイン

コードだけで描かれた柴犬です。SVGパスで体の構造を表現し、CSSアニメーションですべての動きを駆動しています。

カラーリングは本物の柴犬の色：オレンジブラウンのメインコート、クリーム色のお腹と頬、ダークブラウンのシャドウ層。  
首には小さなスカーフを巻いています。

アニメーションの設計原則は**微動**——振れ幅は小さいが常に存在し、静止しているときでも生命感を持たせます：  
呼吸で体が軽く上下し、頭部が呼吸に合わせてごくわずかに傾き、目が自然にまばたきし、しっぽは常にゆっくり揺れています。

歩行は本物の対角歩行（右前・左後が同位相）を採用し、各脚の下半分には独立した膝の曲げアニメーションがあり、着地時に軽い沈み込みの感触があります。

## ツール機能

これらはこの子が飼い主を気にかける方法であり、主な存在目的ではありません：

- **水分補給リマインド** — 定時に小さな犬らしい方法でリマインド。システム通知ではなく。
- **運動リマインド** — 長時間座りすぎると、この子がつついてくる。
- **メモ** — 「覚えておいて：xxx」と言えば、後で聞いたら教えてくれる。
- **一緒にお散歩** — 少し席を離れることを促し、一緒に動く。

## このプロジェクトについて

このプロジェクトの目標はただ一つ：

**仕事中、デスクトップの隅に自分の犬がいる、そう感じてもらうこと。**

だから最も大切なのは機能の多さではなく——自然かどうか、魂があるかどうか、ずっと置いておきたいと思えるかどうか。
